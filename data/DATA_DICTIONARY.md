# Data Dictionary — Synthetic Affiliate Granularity Dataset

All files are **fully synthetic** and **anonymized** (see paper §5). Generated by
`code/generate_dataset.py` with fixed seed `20260623`. Span: 2023-01-01 to
2025-12-31 (1,096 days), 60 brands (`BR-001 … BR-060`).

No file contains a real partner name or a real value.

---

## `brand_registry.csv` (60 rows)

| column | type | description |
|---|---|---|
| `brand_id` | string | Anonymized code, `BR-001` … `BR-060`. |
| `sector` | string | Generic sector tag (Travel, Electronics, Food, …). |
| `reporting_granularity` | string | `daily`, `weekly`, or `monthly`. |
| `role` | string | `calibration` (true daily sales known) or `target` (aggregate only). |

## `behavioral_proxy_daily.csv` (65,760 rows — every brand, every day)

| column | type | description |
|---|---|---|
| `brand_id` | string | Brand code. |
| `date` | date | Calendar day. |
| `proxy_events` | int | Daily behavioral proxy (clicks/sessions/events). The high-frequency indicator. |

## `sales_calibration_daily.csv` (24,112 rows — the 22 calibration brands)

| column | type | description |
|---|---|---|
| `brand_id` | string | Brand code (calibration brands only). |
| `date` | date | Calendar day. |
| `sales_units` | int | True daily units sold (ground truth). |
| `sales_value_eur` | float | True daily revenue (EUR, rescaled). |

## `sales_target_aggregate.csv` (2,832 rows — the 38 target brands)

| column | type | description |
|---|---|---|
| `brand_id` | string | Brand code (target brands only). |
| `granularity` | string | `monthly` or `weekly`. |
| `period_start` | date | First day of the reporting period. |
| `period_end` | date | Last day of the reporting period. |
| `sales_units` | int | Reported units for the period (the only sales figure available for targets). |
| `sales_value_eur` | float | Reported revenue for the period (EUR, rescaled). |

---

## Intended use

- Train/learn the shared proxy→sales elasticity on `sales_calibration_daily.csv`
  joined to `behavioral_proxy_daily.csv`.
- Validate cross-unit transfer by leave-units-out on the calibration brands.
- Apply to `sales_target_aggregate.csv` (+ proxy) to produce a daily panel that
  conserves each reported aggregate.

## Ground-truth parameters (for reference)

The data-generating process uses a shared elasticity **β = 0.88** with per-brand
slack **s_β = 0.05**, lognormal multiplicative noise, weekly + annual
seasonality, and a mild growth trend. These are the values the method aims to
recover.
